# Public Documentation and Testnet Launch Plan

Version: 1.0.0-candidate  
Status: gated plan; launch not authorized

## Phases

1. Freeze identity, schemas, economic policy and exact source commit.
2. Close security, licensing, legal, provider, custody, recovery and capacity reviews.
3. Build reproducible artifacts; generate SBOM/notices/provenance; sign only through approved custody.
4. Rehearse migration, rollback, restore, incident, support and service sunset.
5. Deploy staging and verify every release boolean from direct evidence.
6. Run public-read and bounded value-flow proof with independent observation.
7. Integrate Website copy/metadata/structured data and complete accessibility/localization review.
8. Obtain founder, engineering, security, operations, legal/privacy, economics and brand approvals.
9. Publish only approved surfaces, monitor SLO/guardrails, and retain rollback authority.

## Go/no-go

No-go conditions include unknown artifact/source identity, dirty or unreproducible build, unresolved critical/high security issue, failed restore, missing signer recovery, wrong chain ID, stale or contradictory public evidence, unsupported provider/legal basis, unreviewed economic claim, inaccessible critical flow, missing incident/support routing, or any false release boolean.

## Rollback

Stop new writes first; preserve logs and state; identify accepted actions; use the migration policy; restore only verified artifacts/backups; reconcile uncertain outcomes; communicate impact; and re-open incrementally after authoritative checks. Website claims and download links roll back with runtime status.

## Current decision

No-go. The documentation candidate improves readiness but all nine machine-readable release booleans remain false and multiple external/internal gates are incomplete.
