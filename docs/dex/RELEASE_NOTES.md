# YNX DEX 0.1.0 Testnet Preview candidate

Local candidate adds an independent Web/PWA, immutable constant-product contracts, bounded Router/Quoter behavior, JavaScript SDK, owner-reviewed token-list boundary, persistent Indexer/API with confirmed EVM scanning and reorg recovery, canonical Wallet integration manifest, deterministic live-reserve quotes, Pool Detail and liquidity review, AI risk-explanation workflow, twelve locales, Playwright/offline tests and a deterministic upload-ready PWA bundle.

The final branch additionally adds a non-upgradeable Strategy Vault source candidate. It binds an immutable user owner and limited engine, allows only typed swap/liquidity actions returning assets to the vault, enforces exact approvals, nonce/frequency/capital/gas/deadline/oracle/depeg/loss/drawdown limits, and preserves owner pause/revoke/kill/withdraw/emergency exit. Performance fees are disabled because realized-PnL accounting is not yet available.

The JavaScript SDK adds a limited Execution Adapter. It does not contain keys or an automatic strategy loop: submission requires an exact request digest from canonical Wallet introspection and an explicitly injected engine transport, returns an unconfirmed state, then requires confirmed event reconciliation. Source-labelled gas, public fee, owner-reviewed Oracle and risk observations fail closed against the Vault mandate. CPMM LP fees are truthfully reported as embedded in reserves, so the SDK refuses to fabricate a separate collect transaction. Compound is only an explicitly sized add-liquidity request; rebalance stops after its first request until confirmed reconciliation, fresh state and another Wallet approval.

Indexer schema/cursor v5 retains Strategy Vault, FairFlow and LP Protection ingestion and adds typed Stable Factory discovery. The HMAC cursor binds Vault, FairFlow, LP Protection and Stable Factory addresses plus per-pool version/fee identity. Authenticated schema-v1/v2/v3/v4 state and cursor files migrate only after exact rollback backups; enabling Stable indexing forces a deployment-block rescan. Protected CPMM fees now require an exact `ProtectionAssessed` match instead of assuming the base fee.

FairFlow v1 adds an immutable on-chain Intent and solver state machine. Users retain tokens until atomic settlement; solvers compete through bond-backed commit/reveal solutions scored by verified user surplus at a uniform clearing price. Settlement includes every active Intent, nets Coincidence of Wants, records exact solver external inventory, returns user outputs directly and retains no user tokens. Cancellation after the Intent boundary aborts rather than changing a live competition; non-reveal, timeout and proven same-address self-trade are slashable. The published rebate is solver-funded and is not claimed as attributable MEV without separate proof. The Indexer now decodes all ten lifecycle events and the SDK binds submit/cancel requests to fresh RPC state, nonce domain and exact canonical Wallet approval. UI, cross-chain routes and public Testnet evidence remain absent.

LP Protection adds a separate protected-CPMM Factory whose pools immutably enforce a source-labelled Oracle policy. Dynamic fees disclose base, volatility, trade-depth, Oracle-divergence, directional-flow and recent-liquidity JIT components under a public cap. Stale/invalid Oracle and depeg observations stop swaps, while proportional LP withdrawal remains permissionless. Config changes require a two-day delay. The Indexer now persists and serves fixed source-labelled registration/config/assessment events; the SDK binds authoritative fresh quotes to an exact protected swap and reconciles realized fee separately from zero incentive. These metrics are bounded heuristics, not claims of MEV/LVR elimination, insurance or guaranteed LP outcomes. UI presentation, reviewed live Oracle adapter and public Testnet evidence remain absent.

StableSwap v1 adds a separate clean-room two-asset Factory with immutable decimal normalization, amplification and fee. It supports direct/multi-hop exact input and minimal exact output through dedicated Router/Quoter deployments, proportional LP entry/exit, protocol-fee separation and reserve-ratio accumulators. Schema v5 labels Stable events explicitly and the SDK quotes only from fresh confirmed RPC state. This is not evidence that any asset is stable: external peg/rate Oracles, depeg pause, rebasing/ERC-4626 support, UI transaction review, Vault routing, audit and public Testnet pools remain absent.

This is not a deployed release. The observed public RPC identifies YNX Testnet chain 6423, but no verified contract addresses, reviewed tokens, pools, liquidity, Wallet swap/LP proof, staging/public URL, hosted artifact, production signature, store review or independent audit exists. All corresponding central/deployment/distribution flags remain false.
