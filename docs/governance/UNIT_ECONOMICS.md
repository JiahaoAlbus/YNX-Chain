# Unit economics

Governance does not trade, custody assets, hide spread, mint revenue, burn revenue, or charge performance fees. The local service currently has no user fee and no measured service revenue.

Cost drivers are compute, state and backup storage, App Gateway requests, BFT submission and receipt verification, monitoring/log retention, security review, hardware signer operations, incident response, support, and public archival bandwidth. Provider prices and production traffic are not available, so cost per active voter and gross-margin claims remain unmeasured.

Staging measurement must record monthly active voters, proposals, votes, read requests, state growth, backup bytes, log/metric bytes, compute hours, provider charges, support hours, and any subsidized transaction fees. Report cost per active voter and cost per completed proposal separately. Any future fee requires a public proposal, readable economic impact, explicit user disclosure, parameter bounds, timelock, and auditable receipts.
