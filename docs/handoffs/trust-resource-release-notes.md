# Trust Center and Resource Market 0.2.0 candidate release notes

## Trust Center

- Adds evidence-complete Request Validity with explicit illegal/abusive, overbroad, out-of-scope, insufficient-evidence, governance-review, valid, notice, rejection, appeal, correction, transparency, and audit state.
- Rejects secret extraction, signature bypass, native YNXT freeze/seize/blacklist/transfer, deleted/hidden audit, fake risk, mass tracking, and AI punishment. Reviewers cannot override those boundaries to valid.
- Adds integrity-checked atomic persistence, backup restore CLI, actor-scoped idempotency, strict POST schemas, canonical Wallet session verification, least-privilege AI preview/permission/review, and honest provider failure.
- Replaces the prior left-heavy shell with a compact LTR-first request desk, evidence inspector, folded locale menu, mobile bottom navigation, semantic illegal status, dark mode, and large text.

## Resource Market

- Adds bounded capacity drafts, policies, quotes, signed intents, exact replay, delegation/rental/sponsorship, revocation, dispute recovery, income, analytics, settlement timeline, and audit.
- Separates quote, signed intent, authority acceptance, capacity evidence, and asset settlement proof. No transaction/proof means no settled income.
- Adds the same canonical Wallet, persistence, strict JSON, idempotency, AI permission, recovery, and fail-closed integration boundaries without sharing product sessions.
- Introduces a distinct quote/intent execution console with compact market metrics, folded locale menu, mobile bottom navigation, dark mode, and large text.

This is a local/tested candidate. It is not centrally merged, staged, public, production-signed, store-released, or publicly downloadable.
