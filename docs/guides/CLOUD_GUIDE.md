# YNX Cloud Guide

Status: candidate product guidance; durable public storage service not evidenced

Before upload, a production YNX Cloud must disclose storage provider, region/jurisdiction, encryption and key boundary, sharing permissions, size/rate limits, retention, versioning, deletion/export, offline/conflict behavior, cost and outage state. A provider upload acknowledgement is not durable-recovery proof.

Users should retain independent copies of critical data. Sharing must be least-privilege, expiring and revocable. AI access requires context consent, preview and bounded data selection; it cannot silently publish, delete or broaden access. Current release/download/store states remain false.
