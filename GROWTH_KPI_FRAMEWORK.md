# Founder KPI and Decision Framework

No public telemetry exists, so every current KPI value is unavailable rather than zero. Before collection, publish event schemas, consent/retention rules, bot filtering, source and failure status. Wallet addresses must be pseudonymized for product analytics and never joined to secrets or off-product tracking.

| KPI | Definition | Decision use |
| --- | --- | --- |
| Activation | Distinct eligible users who complete one source-labelled economics review or successful testnet task / eligible new users | Diagnose onboarding truth, not vanity visits |
| 7/30-day retention | Activated cohort returning for a valid testnet task in day windows | Scale only with meaningful repeated use |
| Task completion | Completed valid task attempts / started attempts, split by failure reason | Fix reliability and comprehension |
| Crash-free session | Sessions without process/browser crash / observed sessions | Stop promotion on material regression |
| Support load | Human support minutes and cases / active users | Include in unit cost |
| Abuse rate | rejected or confirmed abusive actions / eligible actions, with appeal corrections | Set limits without punishing legitimate users |
| Provider cost | Invoiced provider units and currency / active user and completed task | Preserve source invoice and free-tier exhaustion |
| Gross margin candidate | Recognized consented revenue less variable service cost / recognized revenue | Undefined while revenue is zero |
| Public Testnet usage | Distinct valid wallets, transactions and economic-page users from public source evidence | Exclude bots, internal probes and faucet loops |
| Conversion | Eligible users entering the explicitly defined next product stage / eligible exposed users | Never use fake urgency or social proof |

Scale only after four consecutive weekly cohorts meet reliability, safety, support and contribution-margin targets set from real baseline data. Continue a bounded test when learning value is positive and safety/cost caps hold. Kill or redesign when abuse or support cannot be bounded, core task completion remains below an approved floor, custody/legal controls are unavailable, gross margin cannot become sustainable without hidden fees/emissions, or users misunderstand candidate economics as guaranteed outcomes.
