# Operations

Start a local isolated candidate with `YNX_RESOURCE_MARKET_DEV_HEADER_AUTH=1` only on loopback. Non-development deployments must omit trusted-header mode and configure the accepted canonical Wallet/App Gateway. Separate product and market store paths must reside on encrypted operator-managed storage.

Preflight: verify exact source commit, clean tree, configuration source, filesystem capacity, backup destination, central registry deployment, provider credentials, clock synchronization and fail-closed health. Run unit, race, smoke, Wallet-vector, capacity, secret, dependency and artifact checks.

Incident priority is asset integrity, user exit and evidence preservation. Disable new quote/reservation traffic while retaining state reads, meter submission needed to close running work, disputes, refunds and export. Never relabel an incomplete service or pending transaction as settled. Record request, audit, meter, order and receipt IDs.

Recovery: stop writers, preserve the primary and backup bytes, validate envelopes, restore to a new path, start read-only verification, compare counts/digests, then reopen writes after operator approval. A successful process restart is not a disaster-recovery drill.

Provider exit requires maintenance state, no new matching, active-order inventory, buyer notice, data/artifact transfer, final meters, settlement/refund, dispute window, export and bond release after liabilities expire. Service termination requires user/provider exports and a funded refund/exit plan before shutdown.
