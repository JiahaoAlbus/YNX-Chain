# Unit economics

No revenue, margin, demand, token price or provider-cost estimate is represented as observed production data.

For each settled meter:

`gross cost = provider cost + protocol fee`

`provider net = evidenced quantity × accepted unit rate − explicit provider-side adjustments`

`YNX contribution candidate = protocol fee − payment cost − gateway cost − storage cost − support/refund reserve − attributable infrastructure cost`

Only an asset-settlement-confirmed receipt contributes to realized service revenue. Quotes, reservations, started services, metered usage, unpaid invoices, token issuance, burn and treasury movement are not revenue. Unrealized user profit is never fee-bearing.

Every quote exposes unit, quantity, rate, provider cost, protocol fee, currency, expiry and source. Refunds reconcile to metered gross cost. Bond penalties compensate proven fraud or SLA violation and are not operating revenue. Subsidies are reported separately and cannot make an uneconomic provider appear profitable.

Before public scale, operators must populate measured cost tables for active-user infrastructure, request processing, meter storage, backup, egress, AI/provider calls, support, dispute review, free allowance and subsidy budget. Scale requires positive contribution after refunds and support at measured usage; kill or reprice if provider cost or abuse makes that impossible.
