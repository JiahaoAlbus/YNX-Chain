# Release notes — 0.3.0 candidate

This unreleased local candidate adds a durable infrastructure market domain: provider verification, offers and matching, sealed reverse and split batch procurement auctions, quote-to-receipt state separation, expiring Ed25519 worker keys and signed metering, reconciliation, provider failure/retry, capped bond penalties and independent appeal. Auction clearing enforces immutable commitments, deadlines, self-dealing rejection, deterministic tie-breaking and honest insufficient-fill failure; output remains an expiring quote, never a reservation or settlement. Provider maintenance, capacity updates, guarded exit/migration, actor-scoped export, independent erasure fulfillment and a versioned transient-record retention sweep are also included. Erasure is blocked by live obligations and pseudonymizes identity without deleting economic, dispute, receipt or audit integrity. Authenticated `/api/market/*` routes, Provider/Buyer Web workspaces, request IDs, structured logs, process metrics and a reproducible local capacity probe are included.

This candidate is not centrally integrated, staged, public, hosted, production-signed or store-released. It has no public transaction receipt. A fresh Android debug APK was installed and cold-started on an Android 16 emulator; this does not prove production signing, native feature parity, canonical callback completion or public authority access.
