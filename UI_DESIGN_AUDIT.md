# YNX Data Fabric UI Design Audit

An embedded read-only operator console is implemented at `/operator/`. It reads real health/version/metrics and uses a canonical Wallet bridge for request-bound, cookie-free, Bearer-free product records. When the bridge is absent it reports permission failure; it never substitutes fixture data or a static success state. No write or recovery control is exposed because an audited preview/approval/confirmation API is not yet complete.

Source and automated checks directly cover semantic landmarks, live regions, tab/table relationships, keyboard focus, reduced motion, light/dark color schemes, fluid type, logical RTL properties, a 390px breakpoint, and loading/success/empty/failure/offline/permission/expired/recovery states. English, 简体中文, 繁體中文, 日本語, 한국어, Español, Français, Deutsch, Português, Русский, العربية and Bahasa Indonesia are selectable; Arabic sets document direction to RTL. Dynamic operational states are translated, while several technical field labels intentionally retain shared terminology in languages whose locale overlays inherit the English base.

Chrome 150 captures now cover the truthful unavailable state at 1440×1200 and an actual 390×1200 CSS-pixel device viewport. The 390px run directly measured document and body scroll width at 390px, so that state has no horizontal overflow. Hashes, bytes, viewport facts and limitations are recorded in `evidence/ui/operator-console-unavailable-state-20260722.json`.

This is not a passed visual-release audit. The captures use a static local asset server with no data fixture, which proves honest failure rendering but not successful API behavior. No browser/assistive-technology matrix, contrast measurement, real canonical Wallet bridge session, physical mobile-device run, or user research exists. DLQ detail and Saga recovery actions remain absent. Before public use, verify every locale and legal/risk phrase with native review; capture empty/loading/success/offline/permission/expired/recovery states from authoritative sources; run keyboard and screen-reader checks; and implement any mutation only with preview, canonical authorization, explicit confirmation, result Audit ID and truthful failure.
